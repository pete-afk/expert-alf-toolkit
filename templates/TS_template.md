## SOP 작성 시 유의사항

1. **케이스 품질 9-point checklist**: 각 케이스에 빈도, 시나리오, 블록 인용 멘트, 도구 경로, UI 경로, 조건 분기, 경고 문구, 완료 확인, Case 간 참조가 포함되었는지 확인
2. **팩트 기반 어투**: "~~ 처리 필요", "~~ 설정 필요" 형태로 기재
3. **구체적으로**: "확인하세요"보다 "주문 번호를 어드민에서 확인하세요"
4. **액션 지향적**: "정책은 ~입니다"보다 "~를 안내하세요"
5. **안내 멘트는 블록 인용(>)으로**: 고객에게 보내는 실제 메시지 수준으로 작성
6. **조건 분기 필수**: 각 케이스에 최소 1개의 IF/THEN 분기
7. **예외 케이스**: 일반적인 케이스뿐만 아니라 예외도 포함
8. **비파괴적 순서**: TS SOP는 최소 침습 → 최대 침습 순서 + 매 단계 확인
