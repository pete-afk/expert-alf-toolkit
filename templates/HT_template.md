## SOP 작성 시 유의사항

1. **케이스 품질 9-point checklist**: 각 케이스에 빈도, 시나리오, 블록 인용 멘트, 도구 경로, UI 경로, 조건 분기, 경고 문구, 완료 확인, Case 간 참조가 포함되었는지 확인
2. **팩트 기반 어투**: "~~ 처리 필요", "~~ 설정 필요" 형태로 기재
3. **설명은 쉽게**: 전문 용어 최소화, 예시 활용
4. **안내 멘트는 블록 인용(>)으로**: 고객에게 보내는 실제 메시지 수준으로 작성
5. **조건 분기 필수**: 각 케이스에 최소 1개의 IF/THEN 분기
6. **실용적**: 실제 응답에 바로 사용 가능한 문구 포함
7. **완전성**: 관련 FAQ, 예외 케이스까지 포함
8. **구조화**: 주제별로 명확히 구분, Case 간 참조로 중복 최소화
